盒子模型：任何容器都可以成为一个盒子，

一个盒子由四部分组成

内容区
内边距（padding）：上下左右单独设置，也可以写合并写法，一个值代表上下左右，两个值代表上下、左右，三个值代表上、左右、下，四个值代表上右下左
边框（border）：上下左右单独设置，也可以写合并写法，border有三个设置项（线条宽度  线条颜色  线条类型），还有一个outline外边框，但是不占盒子模型
外边距（margin）设置同padding，可以设置负数

元素实际的占位大小=内容+padding+border

垂直margin合并问题：上下容器的margin空白同时设置，只会取其中的最大值设置，而不是累加设置

垂直margin失效问题的解决，父元素主要具有以下四个条件，垂直margin就不会失效

1 父元素有内容
2父元素有padding
3 父元素有边框线
4 父元素的定位方式是position:absolute或者float浮动

盒子模型的标准模式和怪异模式：

标准模式：padding和border是向外扩展空间
怪异模式：padding和border是向内占据内容空间

  1. 父元素设置固定高度

  3. 父元素设置overflow:hidden

position定位：

可以通过top left right bottom设置位置
static：默认值，不做任何定位，正常文档输出流
absolute：绝对定位，脱离文档输出流，如果没有父元素，或者父元素没有设置position:absolute 或 relative 或 fixed，那么相对于网页原点定位，反之如果有父元素且父元素设置了position:absolute 或 relative 或 fixed之一，则相对于父元素原点定位，一个元素设置为absolute之后位置不会有任何变化（位置跟随），除非设置top left right bottom
relative：相对定位，正常文档输出流，相对于原来的位置进行偏移
fixed：相对于视口定位，脱离文档输出流

叠放次序：脱离文档输出流在上，如果都脱离文档输出流，下面的优先在上（下面的挡住上面的）可以通过z-index属性手动设置叠放次序：值越大，叠放次序越靠上

动态变化

变换：transform 就是把元素进行平移，旋转，缩放，扭曲等操作
平移：transform:translate(x轴，y轴) 像素和百分比（百分比是相对于自己的宽高）
旋转：transform:rotate(角度) 单位是deg
可以设置transform-origin，设置旋转的原点
缩放：transform:scale(宽度倍数,高度倍数)
扭曲：transform:skew(x角度，y角度)

旋转默认是二维平面旋转（Z轴），X轴和Y轴属于三维旋转，但是只有加上远近度才能体现立体效果，远近度越大立体效果越小，远近度越小，立体感越强，远近度必须加在旋转元素的父元素上！

JavaScript

由网景公司创造，早期叫做LiveScript，后期改名叫做JavaScript，后期IE推出了Jscript，由于浏览器的差异，可能会有兼容性问题

编程语言分类

编译型：常见的C,C++，必须提前编译转化为二进制才能执行
解释型：常见JavaScript,SQL语句，一边编译一边执行

JavaScript是解释性语言，由浏览器执行，是面向过程基于对象的语言
js严格区分大小写，每个语句必须用分号结束

和CSS类似，也有内嵌和外联两种写法

内嵌：在<head>或<body>中加入<script></script>标签，在一个网页中<script>可以出现多次
外联：需要把代码写到一个js文件中，然后在html中<script  src=”xxx”></script>引用，引用外链js的<script>标签内部不能写任何内嵌js，会被忽略

注释： //单行注释（ctrl+/） /* 多行注释 */ ( ctrl+shift+/ )
输出语句:document.write(数据);
和网页正常输出流一起输出的时候，会和其他输出流混合输出
如果单独调用执行，会覆盖掉网页原始内容，重新生成一个新的文档输出刘
对话框提示：window.alert(数据);
控制台输出：console.log(数据)

优先级

逻辑非（优先级最高）>逻辑与>逻辑或

逻辑运算的短路运算：

短路与：第一个表达式如果为false，第二个表达式将会不执行
短路或：第一个表达式如果为true，第二个表达式将会不执行

三目运算符： 表达式 ? 语句1：语句2 表达式为true执行语句1，否则执行语句2

由于IEEE标准的特点，计算机无法精确的存储浮点数字，所以浮点运算一定有误差，解决的办法就是运算前放大浮点数为整数，运算后再还原为浮点数

调用（执行）函数语法：

返回值：是函数执行完毕的一个结果，返回给调用者，没有return返回值，默认返回undefined这个值

作用域

作用域指的是数据可以访问使用的范围
全局作用域：window全局对象的成员，声明在函数外部的变量，作用域是整个script程序，称作全局变量，尽量少使用全局变量，避免全局污染
局部作用域：声明在函数内部的变量，称作局部变量，作用域仅限于当前声明的函数内部，形参也是局部变量

在函数中不声明直接使用的变量，默认就是全局变量
在JS中没有块级作用域这一说 作用域最小范围就是一个函数
作用域链：当在某个function中使用某个变量时，首先会优先使用当前function中的声明原则（就近原则），如果没有，就回到外层（上层）作用域查找使用，直至全局作用域，如果没有，直接报错

原型prototype

每个类型（例如Date Array Number RegExp Person） 都有属性prototype 这个属性指向了这个类型的原型对象，原型对象默认就存在是JS自动创建的一个对象，这个对象默认有两个成员，一个是constructor 默认指向函数本身，一个proto默认指向Object的原型对象

每一个对象实例也都有一个属性，叫做proto，指向的就是这个对象的类的原型对象
xxx类的原型对象 自带了几个
当调用某个对象成员的时候，首先会在当前对象实例中找这个成员，如果有就直接调用，如果没有，则会到当前对象的原型对象中查看有没有，如果有就直接调用

所以如果把方法定义到原型对象中，就不需要每次创建对象重复的创建方法，而是所有的对象访问的都是同一个方法，节省内存

http通信过程：客户端浏览器发出一个url请求，连接服务器，服务求接收请求，请求的如果是一个静态资源（.html .css .js .jpg），服务器会直接产生响应，把请求的资源内容发到客户端，如果请求的动态资源（asp java asp.net ruby python php）就会先在服务器执行后端程序，然后把执行结果响应到客户端

请求常见的两种方式

get请求和post请求 浏览器默认只有一种情况下能发出post请求，必有form表单，且form的method属性值为post,其他请求方式还有put 和 delete

发送数据的格式：url？参数名=参数值&参数名=参数值&……
请求响应，如果服务器没有做过特殊设置，默认编码一律是ISO-8859-1编码（西欧编码）
一个完整的HTTP请求是三部分组成：请求行 请求报头 请求正文

响应的状态码

200 请求响应成功
301 302后端重定向跳转
304打开了缓存过的网页（从浏览器缓存中读取内容）
404请求的资源不存在
500服务器端程序出错
405 get或post请求找不到对应的处理程序

Java中能够被url请求调用的类，是JavaEE中的Servlet
浏览器本身就具备发送请求得到响应的能力，但是浏览器发送普通的请求一定会产生页面跳转，如果需要发送请求得到响应而页面不会发生任何跳转，需要使用ajax技术
在JavaScript，能够发送ajax请求的是XMLHttpRequest对象，需要做兼容，ajax发送请求，默认发送请求是异步请求

请求

同步请求：顺序执行，一个请求执行完毕才能执行下一个请求或其他操作
异步请求：并发执行，多个请求或操作可以同时执行

Jquery提供的ajax方法

$.get() 发送get请求
$.post() 发送post请求
load() 自动把请求响应的结果载入到当前容器中，默认是get请求，如果有请求参数，则自动变为post请求
$.getJSON() 请求响应json格式的数据
$.getScript() 动态加载外联js
$.ajax()最完整的写法

Jquery所有ajax方法都会返回一个对象 这个对象就是当前的ajax请求对象

为了防止攻击，浏览器默认不允许ajax发出跨域请求

解决ajax跨域请求问题的方案

  1. 修改浏览器配置 让浏览器允许跨域ajax
  2. 服务器端代码设置Access-Control-Allow-Origin参数，允许其他域的ajax请求（CROS）
  3. 利用iframe方式解决
  4. 利用Jsonp 方式跨越，浏览器只禁止ajax跨域请求，并不禁止浏览器正常的跨域请求 jsonp就是利用外联其他域的js实现的
    浏览器请求一个url地址，如果发现是以前访问过的url 并且这个对应资源的修改时间没有任何变化，则直接访问浏览器本地缓存，而不是发出新的请求（304）,

Jquery：在jquery的功能上增加额外一些功能

全局插件：定义在$中，通过$调用
局部对象插件：通过JQ对象来调用

HTML5

HTML5就是第五代超文本标记语言，从广义上讲，HTML5不是HTML4的升级，而是HTML，CSS，JavaScript的全面升级，新增了很多标签和API，由于兼容性问题，大多数用于移动端开发。

H5网页的文档声明：

为了解决div在布局中嵌套过多的问题，H5推出新的语义化标签，优点是代码可读性好，做SEO（搜索引擎优化）比较方便

语义化标签

article标签装载显示一个独立的文章内容
section 标签定义文档中的节（section、区段）
aside 用来装载非正文类的内容
hgroup 标签用于对网页或区段的标题元素（h1-h6）进行组合
header 标签定义文档的页面头部，通常是一些引导和导航信息
footer 标签定义 section 或 document 的页脚
nav 标签定义显示导航链接
